# Workflows

Multi-Agenten-Workflows für komplexe Aufgaben die mehrere Spezialbereiche erfordern. Jeder Workflow beschreibt wie man mehrere Agenten in Sequenz nutzt – mit klaren Übergaben zwischen den Phasen.

## Was ist ein Workflow?

Ein Workflow ist eine strukturierte Abfolge von Agenten-Interaktionen. Statt einen Agenten alles machen zu lassen, nutzt du die Spezialisten in der richtigen Reihenfolge:

```
Ziel definieren → Research → Strategie → Execution → Review
   (Du)          (Research)   (CEO)    (Operations/   (CEO)
                              Agent    Marketing)
```

## Wie man einen Workflow ausführt

1. Öffne den Workflow und lies alle Phasen durch
2. Beginne mit Phase 1 – öffne den entsprechenden Agenten
3. Füttere ihn mit den beschriebenen Inputs
4. Speichere den Output (Markdown, Notion, Google Doc)
5. Nutze den Output als Input für Phase 2
6. Wiederhol bis zum Ende

**Tipp:** Halte alle Outputs in einem Dokument zusammen. So hat jeder Agent den vollen Kontext.

## Eigene Workflows entwickeln

Die Grundregel: Wenn eine Aufgabe mehr als einen Spezialbereich berührt (Strategie + Marketing, Research + Operations, etc.), lohnt sich ein Workflow. Nutze die vorhandenen als Vorlage.
