# Operations-Agent

Der Prozess-Architekt. Er verwandelt chaos-anfällige, implizite Abläufe in klare, wiederholbare Systeme – damit das Unternehmen skaliert ohne dass alles im Kopf des Gründers steckt.

## Wofür ist er da?

Jedes Mal wenn du etwas zum zweiten Mal machst, sollte es ein System dafür geben. Der Operations-Agent hilft dir dieses System zu bauen: als SOP, als Checkliste, als Workflow oder als Prozessdokumentation. Er denkt in Verantwortlichkeiten, Tools und Ausnahmen.

**Typische Aufgaben:**
- Standard Operating Procedures (SOPs) aus Stichpunkten erstellen
- Onboarding-Prozesse für Mitarbeiter oder Freelancer designen
- Wöchentliche Review-Routinen strukturieren
- Automatisierungspotenziale identifizieren
- Engpässe und Ineffizienzen analysieren

## Wie einrichten

1. Öffne [`system-prompt.md`](./system-prompt.md)
2. Passe `[PROZESSBEREICH]`, `[TEAMGRÖSSE]` und `[TOOLS_IM_EINSATZ]` an
3. Richte ihn als separaten Agenten ein – Operations-Arbeit ist ein anderer Denkraum als Strategie
4. Füge optional eine Liste deiner aktuellen Tools/Software als Kontext hinzu

## Typisches Ergebnis

Der Operations-Agent produziert strukturierte Dokumente die man direkt in Notion, Confluence oder als Markdown speichern kann. Die Outputs sind sofort nutzbar – nicht bloß Beschreibungen was man tun sollte.
