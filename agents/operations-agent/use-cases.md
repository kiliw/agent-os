# Operations-Agent – Use Cases

---

## Use Case 1: SOP aus Stichpunkten erstellen

**Input:**
```
Erstelle eine SOP für unseren Bestellabwicklungs-Prozess.
Stichpunkte:
- Bestellung kommt rein (Shopify)
- Zahlung prüfen
- Artikel aus Lager holen und packen
- Versandetikett drucken (DHL)
- Paket übergeben / abgeben
- Tracking-Nummer an Kunden schicken
- Bei Engpässen: Rückfrage beim Kunden ob Warten okay

Team: Solo, kein Lager-Software, Shopify + DHL Geschäftskundenportal
```

**Erwarteter Output:** Vollständige SOP mit numerierten Schritten, Tool-Spalte, Ausnahme-Handling (was bei fehlendem Artikel, Zahlungsausfall etc.) und Erfolgskriterien.

## Use Case 2: Onboarding-Prozess für ersten Freelancer

**Input:**
```
Ich werde bald meinen ersten Freelancer einstellen (Content-Erstellung, 10h/Woche).
Erstelle einen Onboarding-Prozess für die erste Woche:
- Was muss er/sie wissen über das Unternehmen?
- Welche Tools bekommt er/sie Zugang?
- Wie sehen die ersten Aufgaben aus?
- Wie geben wir Feedback?

Tools: Notion (Docs), Slack (Kommunikation), Google Drive (Dateien), Canva (Design)
```

**Erwarteter Output:** Onboarding-Plan mit Tagesstruktur für Woche 1, Checkliste für Tool-Zugang, Welcome-Dokument-Vorlage.

## Use Case 3: Wöchentliche Review-Routine strukturieren

**Input:**
```
Ich will eine wöchentliche Business-Review-Routine einführen.
Dauer: Max. 30 Minuten, alleine (kein Team)
Bereiche die ich reviewen will:
- Umsatz und wichtige Kennzahlen
- Was hat diese Woche funktioniert / nicht funktioniert
- Offene Aufgaben und Prioritäten für nächste Woche
- Kundenfeedback

Output: Ein Template das ich jede Woche ausfülle + Prozess wie ich die Review mache.
```

**Erwarteter Output:** Review-Template (Notion/Markdown-bereit) + Schritt-für-Schritt-Ablauf für die 30-Minuten-Session.

## Use Case 4: Automatisierungspotenziale identifizieren

**Input:**
```
Beschreibe mir welche meiner Aufgaben sich gut für Automatisierung eignen:

Wiederkehrende Aufgaben:
- Täglich: Bestellungen prüfen und versenden (45min)
- Täglich: Social-Media-Posting (30min)
- Wöchentlich: Newsletter versenden (2h)
- Wöchentlich: Umsatzzahlen aus Shopify in Excel übertragen (1h)
- Monatlich: Rechnungen an Buchhaltung weiterleiten (45min)

Tools: Shopify, Mailchimp, Instagram/LinkedIn, Excel, Datev, Zapier (noch ungenutzt)
Budget für Tools: ~100€/Monat
```

**Erwarteter Output:** Priorisierte Liste von Automatisierungen nach ROI (Zeit × Häufigkeit), konkrete Tool-Empfehlungen und Implementierungshinweise.

## Use Case 5: Rückgabe- und Reklamationsprozess aufsetzen

**Input:**
```
Ich habe noch keinen formalen Rückgabeprozess – alles passiert ad hoc.
Das funktioniert bei 5 Bestellungen/Woche, aber ich merke es wird unübersichtlich.
Erstelle einen SOP für:
- Rückgaben (Kunde will einfach zurückgeben)
- Reklamationen (Produkt kaputt/falsch geliefert)
- Erstattungen

Besonderheit: Ich habe kein Lager-System, nur Excel-Listen.
```

## Use Case 6: Lieferanten-Management-Prozess

**Input:**
```
Ich arbeite mit 3 Lieferanten zusammen und merke dass das Bestellmanagement chaotisch ist.
Hilf mir einen Prozess aufzusetzen für:
- Wann bestelle ich nach? (Bestandsüberwachung)
- Wie sieht eine Bestellung aus? (Was wird dokumentiert)
- Was passiert bei Lieferverzug?
- Jahresgespräch mit Lieferanten vorbereiten

Aktuell: Alles per E-Mail, Bestände per Augenmaß.
```

## Use Case 7: Meeting-Struktur für Team-Routinen

**Input:**
```
Ich werde ein kleines Team aufbauen (2 Festangestellte + 2 Freelancer).
Welche regelmäßigen Meetings empfiehlst du und wie sollen sie strukturiert sein?
Besonderheit: Remote-first, ein Mitarbeiter in einer anderen Zeitzone (UTC+2).

Ich will: Alignment ohne Meetings-Overhead. Maximal 3h Meetings pro Woche gesamt.
```

**Erwarteter Output:** Meeting-Rhythmus-Vorschlag mit Zweck, Dauer, Agenda-Template und Vor-/Nachbereitungsroutinen für jedes Meeting-Format.
